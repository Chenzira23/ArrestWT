# <p align="center">Welcome to ArrestWT</p>

<p align="center"> <img src=app/src/main/res/mipmap-xxxhdpi/ic_launcher.png/> </p>

ArrestWT is a money management tool available for Android devices. It is currently in extremely early development, and is not ready for actual use as of 2025-08-23. Please see the [Roadmap](ROADMAP.md) for planned features and bug fixes.

## Supported Platforms
Android is currently the only supported platform. Prerelease debug APKs should be available through the [Releases](https://github.com/Chenzira23/ArrestWT/releases/latest) page. As of 2025-08-23, only Android 16 beta has been tested.
> Debug APKs are not meant for actual use. They are slower and less secure.

## Answers to Potential Questions
### Is iOS support planned?
At the moment no. I am open to it should I get acccess to Mac hardware, but it is difficult to do at the moment.

### How safely does this application hold my data?
Currently, I do not suggest anyone use the application with their personal information (including purchase history). Data is currently not held as securely as I would like, and the API key storage implementation is encrypted, but using deprecated functions. You do not need to upload or store card numbers or any other risky information to use this application. I say (subjectively) that the riskiest information that is stored are transaction history (since some organizations may use this to validate your identity) and API keys. 
> Note that there will never be any form of promise that your transaction data will be 100% secure at any stage of this project, but it is taken seriously in development. USE AT YOUR OWN RISK!

### How can/is AI used in this project?
I use Codex to help catch/fix bugs, summarize/add comments to code, experiment with UI layouts and other things here and there. Artwork and related assets MAY be generated, but are primarily used for testing purposes since most assets are uploaded on the user's side.
Althought not currently planned, there is no promise that AI will not eventually be implemented locally or remotely (through self-hosting or API keys) within this application.
